# セキュリティ方針

このプロジェクトはユーザーとコントリビューターの安全を重視しています。脆弱性を発見した場合、一般公開の Issue/PR ではなく、GitHub の Security Advisories 機能を利用して報告してください。

- 公開 Issue/PR に脆弱性の詳細を書かないでください。
- GitHub の「Report a vulnerability」から、非公開のセキュリティ勧告として報告できます。
- メンテナは可能な限り迅速に triage・修正・公開を行います。
- 連絡先メールアドレスは公開しません（秘匿）。必要なやり取りは GitHub 上の非公開チャネルで行います。

## サポート対象
- 現在サポート対象は最新の `main` ブランチおよび最新リリースです。
- 古いリリース系列のセキュリティ修正は状況によりベストエフォートとなります。

## セキュリティアップデートの公開
- 影響範囲の評価、修正、検証の後、CHANGELOG に記載しリリースします。
- 必要に応じて回避策（mitigation）も併記します。
