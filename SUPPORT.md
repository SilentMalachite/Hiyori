# サポートポリシー

問題が発生した場合は、以下の手順に従ってください。

1. 既存の Issue を検索
   - 同様の報告や既知の問題がないか確認してください。

2. ログの取得
   - `logs/hiyori.log`（存在する場合）やコンソール出力の該当部分をコピーしてください。

3. 最小再現手順を作成
   - できる限り簡潔なステップで再現できる状態にしてください。

4. Issue を作成
   - 「バグ報告」テンプレートを選び、必要項目を記入してください。

5. 機能要望
   - 「機能要望」テンプレートを選び、背景/解決したい課題/提案内容を記載してください。

## セキュリティに関する報告
- 公開の Issue/PR に脆弱性の詳細は記載しないでください。
- GitHub の Security Advisories（非公開）から報告してください。詳細は `SECURITY.md` を参照してください。

## サポート範囲
- 最新の `main` と最新リリースを優先してサポートします。
- 古いリリース系列はベストエフォート対応となります。

## よくある質問（FAQ）
- テストがヘッドレス環境で失敗する
  - `-Djava.awt.headless=true -Dhiyori.headless=true` を付与してください。
- CIでタイムアウト/リソース不足
  - テスト並列を下げる（`TEST_MAX_PARALLEL_FORKS=2`）/ DBプールを小さくする（`DB_POOL_SIZE=4`）。
